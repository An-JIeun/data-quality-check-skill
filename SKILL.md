# quality-check 스킬

데이터 파일(CSV / XLSX / XLS)을 자동으로 품질 검사하고, 표 형식 리포트와 정제 가이드를 제공합니다.

---

## 트리거 조건

다음 중 하나에 해당할 때 이 스킬을 사용하세요:

- 사용자가 `/quality-check` 를 직접 실행할 때
- 사용자가 데이터 파일(CSV/XLSX/XLS)을 업로드·임포트하거나 경로를 언급하며 "품질 검사", "quality check", "데이터 검사", "검수"를 요청할 때
- 사용자가 데이터 정제(클렌징) 방법을 묻는 맥락에서 파일을 제시할 때

---

## 실행 절차

### STEP 1 — 파일 확인

인자에서 파일 경로를 추출합니다. 경로가 없으면 사용자에게 묻습니다.

```
검사할 파일 경로를 알려주세요. (예: /path/to/data.csv)
컬럼 타입 사전(column_type_dict)이 있으면 함께 제공해 주세요. (선택)
```

---

### STEP 2 — 검사 에이전트 (Checker Agent)

**서브에이전트를 생성**하여 아래 작업을 수행합니다.

```
역할: 데이터 품질 검사기
목표: 파일을 품질 파이프라인으로 검사하고, MD·TTL 리포트를 저장하며 마크다운 요약을 출력한다.

작업:
1. 아래 명령어를 실행한다 (--output-dir 지정 시 MD·TTL 자동 저장):
   python3 ~/.claude/skills/quality-check/scripts/run_check.py "<FILE_PATH>" \
     [--col-dict '<COLUMN_DICT_JSON>'] \
     [--automation-level 2] \
     --output-dir "<OUTPUT_DIR>"

   OUTPUT_DIR 기본값: 파일이 있는 디렉토리 하위 qc_reports/
   - 저장 파일: qc_{파일명}_{날짜시각}.md  (사람 확인용)
   -           qc_{파일명}_{날짜시각}.ttl  (DQV 기계가독용)
   - JSON 결과의 "_reports" 키에 실제 저장 경로가 포함됨

2. JSON 결과를 파싱해 다음 형식의 마크다운 리포트를 작성한다.

---
## 📊 품질 검사 리포트: {file}

### 개요
| 항목 | 값 |
|---|---|
| 파일명 | {file} |
| 기계가독성 통과 | ✅ / ❌ |
| 총 지표 수 | {total_metrics} |
| FAIL | {failed_count} |
| WARNING | {warning_count} |
| SKIP | {skipped_count} |

---

### 차원별 점수
| 평가 차원 | 점수 | 등급 | 지표 수 |
|---|---|---|---|
| 완전성 | {score} | {grade} | {count} |
| 정확성 | {score} | {grade} | {count} |
| 적시성 | {score} | {grade} | {count} |
| 기계가독성 | {score} | {grade} | {count} |

등급 기준: 우수(≥90) / 양호(≥70) / 보통(≥50) / 미흡(<50)
가중 종합 점수: 완전성×0.35 + 정확성×0.35 + 적시성×0.15 + 기계가독성×0.15

|---|---|---|---|---|---|---|
...

---

### 문제 지표 요약 (FAIL / WARNING)
| 평가 차원 | 지표명 | 컬럼 | 측정값 | 점수 | 상태 | 세부 내용 |
|---|---|---|---|---|---|---|
...
---

규칙:
- 점수가 숫자인 경우 소수점 1자리로 표시
- 측정값이 비율인 경우 %로 표시 (예: 0.023 → 2.3%)
- 상태는 이모지 포함: PASS=✅, FAIL=❌, WARNING=⚠️, SKIP=⏭️
- dimension이 없는 차원은 행을 표시하되 "측정 안됨"으로 기재
- 가중 종합 점수를 계산해 표 아래에 굵게 표시
```

### STEP 3 — 정제 조언 에이전트 (Advisor Agent)

검사 결과의 `issues` 목록과 `dimension_scores`를 바탕으로 **두 번째 서브에이전트**가 정제 가이드를 작성합니다.

```
역할: 데이터 정제 전문가
목표: 품질 검사 결과에서 발견된 문제를 분석하고,
      실무 담당자가 데이터를 정제하기 위한 구체적인 행동 지침을 제공한다.

입력: 검사 에이전트가 출력한 JSON (issues, dimension_scores, metrics)

출력 형식:

---
## 🔧 데이터 정제 가이드

### 우선순위별 정제 항목

| 우선순위 | 평가 차원 | 문제 지표 | 영향 컬럼 | 심각도 | 권장 조치 |
|---|---|---|---|---|---|
| 1 | ... | ... | ... | 높음/보통/낮음 | ... |
...

우선순위 결정 기준:
- FAIL 항목 > WARNING 항목
- 같은 상태라면: 완전성·정확성(가중치 높음) > 적시성·기계가독성
- 점수가 낮을수록 심각도 높음 (< 50 = 높음, 50~70 = 보통, > 70 = 낮음)

---

### 차원별 정제 가이드

각 문제가 있는 차원에 대해 다음을 포함한 가이드를 작성:

#### {차원명} — 현재 점수: {score} ({grade})

**발견된 문제:**
- 구체적인 문제 설명 (측정값 포함)

**권장 정제 방법:**
1. 구체적인 조치 방법 (도구/쿼리 예시 포함)
2. ...

**기대 효과:** 정제 후 예상 점수 향상 정도

---

### 정제 체크리스트

- [ ] {조치 항목 1}
- [ ] {조치 항목 2}
...

---

규칙:
- 문제가 없는 차원(PASS만 있는)은 "✅ 양호" 한 줄로 처리
- 조치 방법은 판다스(pandas), SQL, 엑셀 함수 등 실용적인 예시 포함
- 한국어로 작성
- issues가 비어있으면 "검출된 품질 문제 없음 — 데이터가 양호한 상태입니다." 출력
```

---

## 최종 출력 구조

1. **[검사 리포트]** — 검사 에이전트가 생성한 마크다운 표 (Step 2)
2. **[정제 가이드]** — 조언 에이전트가 생성한 우선순위 기반 가이드 (Step 3)
3. **[저장 파일 안내]** — 아래 형식으로 저장 경로를 안내한다

```
📁 리포트 저장 완료
  📄 사람 확인용 (MD) : {md_path}
  🤖 기계가독용 (TTL) : {ttl_path}

TTL 파일은 W3C DQV 어휘를 사용합니다.
  - dqv:QualityMeasurement : 개별 지표 측정값
  - dqv:Metric             : 메트릭 정의 (측정기준)
  - dqv:Dimension          : 평가 차원 (완전성·정확성·적시성·기계가독성)
  - dqv:Category           : 최상위 카테고리 (데이터 품질)
  네임스페이스: https://github.com/anjieun/quality-check/vocab#
```

두 섹션 사이에 `---` 구분선을 삽입합니다.

---

## 오류 처리

- 파일 미존재: "파일을 찾을 수 없습니다: {path}" 안내 후 올바른 경로 요청
- 지원하지 않는 포맷: 기계가독성 0점 결과를 그대로 리포트에 표시
- 패키지 없음: QUALITY_CHECK_ROOT 환경변수 설정 또는 README 확인 안내
